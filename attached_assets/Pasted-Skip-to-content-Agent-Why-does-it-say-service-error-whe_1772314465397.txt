Why does it say service error when I use the pair code

$ cat /tmp/logs/Start_Application_20260228_213102_026.log
The "service error" occurred because the website was looking for a property named code in the server's response, but the server was actually sending it as pairingCode.

I've fixed this by:

Updating the website code: I corrected the frontend to correctly read the pairingCode from the server.
Improving Error Visibility: I added detailed error logging to both the server and the website. This will help identify if there are other underlying issues (like network or permission problems) by showing the specific error message instead of a generic "Service Error".
Ensuring Directory Permissions: I made sure the routes/session folder exists and has
the correct permissions to store temporary connection data.

The pairing service should now correctly display your code when you enter your phone number.
